Overview

Postgres Database based tool for processing griptester readings.

-fits grip tester readings to network to make table fitted

-Uses benchmark runs and sections to correct for season

-Splits requested sections of network into 10m pieces and finds sfc value from fitted to make table to resized

-Makes hmdif files from table resized.

How to Use

Under 'setup':

-connect to a database

-setup the database if this has not already been done.

-the network is in table -network. Import this into qgis then set network layer to this.

-readings are in table -r. Import this into qgis then set readings layer to this.

under 'upload':

-upload the readings

-ensure information in run_info table is correct

Under 'fitting':

Fit readings to network.(see fitting )

Under 'output':

Download hmd files.

Under 'coverage':

under requested table:

-use 'search hmds' button to find sections missing from the hmd files. Check table r on the map to make sure the section wasn't surveyed .Then look for a reason on the map or in the summary sheet.

-update the note column with the reason.

Use the tables in "View Lengths" and "View Missing Sections" to make the submission sheet.
