Fitting

The routes table is used to fit readings to a section,direction,xsp and section chainage. The fitting tab contains tools for editing it.

Routes table

The routes table contains the following columns

  • sec:section label. Needs to be in network
  • reversed: if survey is in opposite direction to section.
  • xsp
  • run:needs to be in run_info.
  • s,e=f_line where run enters/leaves section.
  • note=note for benefit of person fitting. Autofit adds rows with note of 'auto'.
  • pk=primary key.
  • start_sec_ch,end_sec_ch: Section chainages(in direction of section) where run enters/leaves section. Can be used to calculate section chainage.

Readings in run with s<=f_line<=e are fit to sec,reversed,xsp

The notes column is for the benefit of the person fitting. Autofit will insert rows with a note of 'auto'.

Files sometimes contain repeated chainages. f_line is used for fitting because it is unique(within the run) and increases in direction of survey.

Section Chainage

Section chainage is usually calculated from the nearest point on the section to the reading.

Where start_sec_ch and end_sec_ch are set and >0 section chainage is calculated by interpolating between these.

This should be done where the nearest point of the section is not at the correct chainage.

This is important where the GPS has lost signal and for small roundabouts (that readings are closer to wrong side of).

Autofit

Click the autofit button to automatically create the route for the selected run. It is likely to make mistakes where:

  • The nearest section to a reading is not the correct one
  • The gps is inaccurate
  • The network is inaccurate

Autofit does not work out xsp. Do this by manually or with update statement (update routes set xsp='CL1' where not reversed; update routes set xsp='CR1' where reversed;).

Autofit adds rows for each section near to and in the correct direction for the readings.

How to use

  • Autofit the route.
  • Manually remove any mistakes.
  • Rows with a yellow background have have a gap and should be checked.
  • Rows with a red background overlap by more than 1 reading and are probably wrong.
  • Right clicking a row then "select on layers" is a quick way to check a row.
  • Process the run.
